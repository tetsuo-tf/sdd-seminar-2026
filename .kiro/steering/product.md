# Product Overview

社内向け **IT 問合せ管理システム**。情報システム部門に集まる問合せをアプリ内で受付・追跡・対応するための社内ツール。これまでメール・チャット・口頭に分散していた経路を一本化し、社員と情シス担当の双方が「いま何件、何が、どの状態にあるか」を一覧で把握できる状態を目指す。

## Core Capabilities

- **問合せ登録**: 社員がフォーム（タイトル / カテゴリ / 本文）から問合せを登録
- **役割別の一覧**: 社員は自分の問合せのみ、情シス担当は全件を閲覧
- **検索 / フィルタ / ソート**: キーワード、ステータス、カテゴリ、登録日時で絞り込み
- **ステータス管理**: 「受付済 / 対応中 / 完了」の手動更新（情シス担当のみ）
- **認証 / 認可**: email + password による認証、`employee` / `it-staff` の 2 ロール

## Target Use Cases

- 社員が自分の出した問合せの状態を自分で確認したい
- 情シス担当が全社の問合せを一覧して優先順位を付けたい
- 受付漏れ・対応漏れを減らしたい

## Value Proposition

- **シンプルさ優先**: SLA・自動割当・通知・ナレッジ化などは持たず、「受付と一覧」に機能を絞る
- **社内・少人数利用前提**: SQLite + Better Auth の構成で運用負荷を最小化
- **日本語 UI 専用**: ラベル・メッセージ・バリデーションエラーは日本語で統一

## Out of Scope（本プロダクトでは持たない）

- 添付ファイル、メール / チャット通知、SSO、多言語化
- SLA、エスカレーション、自動割当、担当者アサイン
- コメントスレッド、対応履歴の差分管理、監査ログ
- 多テナント（複数会社対応）

---

_スコープを意図的に絞っているため、新機能提案時は「受付と一覧」というコアミッションに沿うかを最初に確認すること。_
